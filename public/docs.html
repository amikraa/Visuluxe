Authentication

The API uses API keys to authenticate requests. You can view and manage your API keys in the Dashboard.

Security Notice

Authentication requests must be made over HTTPS. Calls made over plain HTTP will fail.

Pass your API key in the Authorization header with the Bearer scheme.

Terminal 
curl "https://api.ai-image.com/v1/models" \
  -H "Authorization: Bearer YOUR_API_KEY"
Authentication Header 
Authorization: Bearer sk-proj-123456789...

Text to Image

POST https://api.ai-image.com /v1/images

Generate stunning, high-quality visuals from simple text prompts using our state-of-the-art diffusion models.

Parameters

prompt string Required

A text description of the desired image(s). The maximum length is 1000 characters.

model string Optional

The model to use for generation. Defaults to gen-v2.

size string Optional

The size of the generated images. Must be one of 256x256, 512x512, or 1024x1024.

n integer Optional

The number of images to generate. Must be between 1 and 4.

Example Request

cURL 
curl https://api.ai-platform.com/v1/images/gen \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -d '{
    "prompt": "A futuristic city",
    "n": 1,
    "size": "1024x1024"
  }'

Response

    {
      "url": "https://cdn.visionary.ai/generations/gen_abc123xyz.png",
      "seed": 48293
    }
  ]
}

Image to Image

Edit, enhance, or transform an existing image using a descriptive text prompt. This feature allows you to keep the core structure of an image while applying creative or stylistic changes through AI.

POST https://api.ai-image.com /v1/edits

Parameters

Field Type Description
image file The image to edit. Must be a valid PNG or JPEG. Max 4MB.
prompt string Description of the desired output.
strength float Value between 0.0 and 1.0. Higher values allow more variation.

Example Request

Python 
import requests

url = "https://api.ai-image.com/v1/edits"
files = {"image": open("original.png", "rb")}

data = {
  "prompt": "make it winter season, snow",
  "strength": 0.7
}

headers = {"Authorization": "Bearer YOUR_KEY"}

response = requests.post(
  url, headers=headers, files=files, data=data
)

Errors

⚡

Error Semantics for Image Generation

Errors indicate the exact stage at which an image request failed — from input validation to model execution to delivery. This separation allows clients to implement accurate retries, fallbacks, and monitoring.

Handling Rate Limits (429)

When a 429 error is returned, clients should apply exponential backoff or retry after the window resets.

Code Meaning
200OK – Success
400Bad Request – Invalid parameters
401Unauthorized – Invalid API Key
429Too Many Requests – Rate limit exceeded
500Internal Server Error

Security & Privacy

  • Data Isolation: Each request is handled in an isolated execution context to prevent cross-user access or data leakage.
  • Controlled Delivery: Generated outputs are returned directly to the requester and are not indexed, cached, or reused.
  • Request Integrity: All requests are validated and monitored to prevent abuse, tampering, or unauthorized usage.
  • Platform Security: The service operates on hardened Cloudflare and AWS infrastructure with continuous monitoring.

No Persistent Storage

Generated images are stored only long enough to complete delivery and are automatically deleted afterward.

Time-Limited Access

Signed URLs restrict access to generated images and automatically expire for added security.

  • Encrypted Transport: API requests and responses are protected in transit using modern TLS encryption.
  • Access Controls: API keys enforce usage boundaries and can be rotated, restricted, or revoked instantly.
  • No Training: Customer inputs and outputs are excluded from any model training or improvement processes.
  • DDoS Protection: Automated traffic filtering and rate controls help protect against abuse and service disruption.
